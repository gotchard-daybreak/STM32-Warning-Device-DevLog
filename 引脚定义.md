1. 优先级最高：避免冲突 + 保持硬件外设的“最佳引脚”
STM32F103C8T6 的每个外设都有“推荐引脚”（硬件复用功能最完整、驱动最稳、库支持最好），我优先把它们固定下来，避免后期改动麻烦。

OLED（I2C1） → PB8 SCL / PB9 SDA
这是 STM32 官方手册里 I2C1 的唯一标准复用引脚（AF_OD 模式），用其他引脚只能软件模拟 I2C（慢、容易出问题）。你之前指定了 PB8/9，我就没动它。
ESP32 串口（USART1） → PA9 TX / PA10 RX
USART1 是 STM32 最稳定的调试串口（bootloader、printf 重定向都默认用它），硬件支持最好，波特率高（115200+）不丢包。改其他串口容易出问题（比如 USART2/3 有时钟或中断冲突）。
SYN6288（USART2_TX） → PA2
SYN6288 只需单向发送（TX），USART2 独立，不和 USART1 冲突。PA2 是 USART2 的标准 TX 引脚，驱动简单。
W25Q64（SPI1） → PA5 SCK / PA7 MOSI / PA6 MISO
这是 STM32 SPI1 的硬件标准引脚（硬件 SPI 速度快、稳定），CS 用 PB3（空闲且常用作片选）。

2. 第二优先级：中断支持 + 连续性（方便焊接）

编码器（A/B/KEY） → PB0 / PB1 / PB2
PB0–PB2 是 STM32 支持外部中断（EXTI）的经典组合，消抖和方向判断最稳。
而且 PB 口连续排列，杜邦线接起来方便，不容易接错。
DS1302（CLK/DAT/RST） → PB13 / PB14 / PB15
连续三个 PB 引脚，焊线/杜邦线时一眼就能对齐，减少接错概率。
这些引脚普通 GPIO 功能全，不被其他外设占用。
蜂鸣器 → PB10
PB10 离其他信号远，PWM 或 GPIO 驱动都没问题，避免信号串扰。

3. 第三优先级：引脚富裕 + 易扩展

所有分配都集中在 PAx 和 PBx 的常用段（PA0–PA10、PB0–PB15），这些引脚在蓝丸子板子上焊盘最大、最好找。
留下了大量空闲引脚（PA0/1/3/4/8/11–15、PB4–7/11–12、PC13–15），以后加 LED、按键、传感器、调试口都随便用。
没有用 PC 口（PC13–15 常被用作 BOOT 或低速 GPIO），避免兼容性问题。

4. 为什么不乱分配？（避免的坑）
如果不按这个原则分配，很容易踩坑：

把 I2C 接到普通 GPIO → 只能软件模拟，速度慢、CPU 占用高、容易被中断打断。
把串口接到不稳定的引脚 → ESP32 通信丢包、SYN6288 播报卡顿。
把中断信号接到不支持 EXTI 的引脚 → 编码器转动检测不灵敏。
引脚太分散 → 杜邦线乱七八糟，调试时容易短路或接触不良。
